# 手语手套 · Android 手机版

手机直连手套蓝牙（JDY-18 BLE），中文手势词→可选 DeepSeek 组句→语音播报；
生命体征异常时**持续响铃+震动+全屏告警（手动才停）**，并把**求助消息+手机 GPS 定位**推送给家人微信。

## 功能
- BLE 连接已配对的 JDY-18，接收 `GESTURE:<name>` 手势
- 蓝牙连接成功后，设备列表只显示并锁定当前已连接设备；主动断开后才恢复完整的已配对设备列表
- 手势识别提供独立总开关；关闭后保持蓝牙连接，但忽略所有手势、停止组句且不触发 SOS 手势告警
- 固件当前 37 类词表完整映射为中文手势词，也兼容旧版 6 类和直接发送中文标签；未知英文标签会在界面提示，不再静默丢弃
- 收到手势后立即更新主显示区，不再在停顿组句期间保留上一次识别词
- 手势词→停顿断句→连词成句；单个手势词原样显示，两个及以上词才调用 DeepSeek，避免云端改写正确词义；API 可独立开启/关闭，内置提示词可在设置页修改
- 中文 TTS 语音播报
- 生命体征卡片（现为模拟数据，真实 MAX30102/GSR 接入后替换）+ 异常自动检测
- **SOS 手势**：独立开关开启时，识别到 `SOS` / `sos` / `求救` 后无需确认，立即持续响铃、震动、全屏红屏并发送定位求助；关闭时“求救”作为普通手语词进入组句
- **生命体征异常**：8 秒可取消倒计时后告警并发送定位求助
- 定位兼容：Google 融合定位不可用时，自动回退到 Android 系统网络/GPS/融合定位；仍无结果时使用 30 分钟内的最新缓存
- 设置页：Server酱 SendKey / 企业微信 Webhook / DeepSeek 开关、Key、提示词 / 称呼 / 停顿秒数（仅存手机本地）

## 怎么编译出 APK（本机无需 Android 环境）

用 **GitHub Actions 云编译**：
1. 新建一个 GitHub 仓库，把本目录（`jdy_gesture_android/`）作为仓库根推上去。
2. 推送后 `.github/workflows/android.yml` 自动运行，编译 debug APK。
3. 打开仓库 **Actions** → 最新一次运行 → **Artifacts** → 下载 `SignGlove-debug-apk` → 解压得 `app-debug.apk`。
4. 传到 Android 手机安装（需在系统里允许「安装未知来源应用」）。

> 也可用 Android Studio 直接打开本目录编译运行。

## 手机上使用
1. 安装后首次打开，授予**蓝牙、定位、通知**权限。
2. 手机系统设置 → 蓝牙 → 与 **JDY-18 配对**（PIN **000000**）。
3. 回 App → 下拉选中 JDY-18 → 「🔌 连接」。
4. 「⚙️ 设置」填入 Server酱 SendKey 或企业微信 Webhook；如需云端组句，再开启 DeepSeek 并填写 API Key。
5. 测试报警：「⚠️ 模拟生命体征异常」→ 8 秒倒计时 → 手机持续响铃+震动+全屏告警 → 点「停止」才停 → 家人微信收到含定位链接的求助。

Android 12 及以上首次启动时，App 会在同一批权限请求中同时申请“精确位置”和“大致位置”，请按实际定位需求授权。

当前 37 类固件词表为：静息、你好、谢谢、再见、我、你、要、喝水、吃饭、厕所、帮助、疼痛、家、求救、走、过来、停、对不起、请、你们、为什么、哪里、怎么样、老师、去、看、评委、数字 0/1/4/7/8/9、欢迎、下午好、很、想。

## 平板定位说明

部分仅支持 Wi-Fi 的平板没有独立 GNSS。请开启系统“位置信息”、WLAN 和蓝牙扫描；App 会优先尝试 Google 融合定位，国行系统无 Google 服务时直接改用 Android 系统定位。求助消息会附带定位来源、精度和缓存时间；完全无法定位时会给出权限、系统定位开关、定位提供程序或超时等具体诊断，但不会阻止告警和求助消息发送。

## 国产 ROM 注意
后台持续响铃/定位可能被省电策略限制，请在系统里给本 App 开：**自启动、后台运行、通知、锁屏显示、定位"始终允许"**白名单。

## 技术
原生 Kotlin · minSdk 26(Android 8) · OkHttp · FusedLocation · 前台服务持续告警 · 定位用高德 uri 链接（免 key，微信可点开）。
